# Paper AI Detector

学术论文 AI 检测与分析 Skill。用于快速评估论文质量、检测 AI 生成内容、识别潜在问题。

## 功能模块

| 编号 | 功能 | 说明 |
|------|------|------|
| 1 | 快速分析 | 结构、摘要、方法、结论概览 |
| 2 | 创新性检测 | 与现有研究的差异化评估 |
| 3 | 文献质量 | 引用可靠性、时效性、相关性 |
| 4 | 语法检查 | 英文写作规范性 |
| 5 | 论文查重 | 与已知来源的相似度检测 |
| 6 | 图片AI检测 | 识别论文中的 AI 生成图片 |
| 7 | AI写作检测 | 文字部分的 AI 生成概率评估 |

## 执行流程

### Step 1: 获取论文内容

优先使用以下方式获取论文：
1. **PDF文件**：读取文件内容，提取文本
2. **文本内容**：直接处理用户提供的文字
3. **URL/链接**：使用 WebFetch 抓取在线论文

```python
# 伪代码：论文内容获取优先级
if has_pdf_file:
    content = extract_text_from_pdf(pdf_path)
elif has_text:
    content = raw_text
elif has_url:
    content = fetch_and_extract(url)
```

### Step 2: 执行检测分析

根据用户需求调用对应模块：

#### 2.1 快速分析 (Quick Analysis)
```
分析要点：
- 论文结构完整性（摘要/引言/方法/结果/讨论/结论）
- 研究问题是否清晰
- 方法论是否合理
- 数据/证据是否充分
- 结论是否有据可依
```

#### 2.2 创新性检测 (Innovation Check)
```
评估维度：
- 研究问题新颖度（与现有文献对比）
- 方法论创新点
- 数据集/实验设计独特性
- 理论贡献度
- 实际应用价值

判断标准：
- 高度创新：全新问题/方法/理论框架
- 中度创新：现有方法的改进/组合/新应用
- 略有创新：增量式改进
- 缺乏创新：重复性研究
```

#### 2.3 文献质量检查 (Literature Quality)
```
检查清单：
□ 引用文献的权威性（顶刊/会议 vs 普通期刊）
□ 文献时效性（近3年占比，是否遗漏重要新文献）
□ 引用相关性（是否精准引用核心文献）
□ 引用完整性（是否遗漏关键反对意见）
□ 引用格式规范性
□ Self-citation 比例是否过高
```

#### 2.4 语法检查 (Grammar Check)
```
英文论文语法检测：
- 主谓一致
- 冠词使用
- 时态统一性
- 介词搭配
- 句子结构复杂度
- 学术写作规范性

输出格式：
[疑似错误位置] 原文 → 建议修改
```

#### 2.5 论文查重 (Plagiarism Check)
```
检测策略：
1. 与已知学术数据库对比（需用户提供参考来源）
2. 段落相似度分析
3. 常用模板句识别
4. 引用覆盖率检查

输出指标：
- 总体相似度百分比
- 高相似度段落列表
- 可疑来源（若用户提供）
```

#### 2.6 图片AI检测 (AI Image Detection)
```
检测方法：
1. 视觉特征分析（过于完美/不合理细节/风格一致性）
2. 元数据检查（EXIF信息）
3. 已知AI图片指纹匹配

警告信号：
- 图片过于高清/规整
- 细胞/组织图缺乏自然变异
- 实验数据图过于完美匹配理论曲线
- 统计图无合理的误差范围

注：如有具体图片文件，使用 AI 图片检测工具进行验证
```

#### 2.7 AI写作检测 (AI Writing Detection)
```
检测维度：
- 词汇丰富度（AI倾向使用高级词汇）
- 句子长度分布（AI倾向等长句）
- 逻辑连接词使用模式
- 段落结构规律性
- 参考文献部分是否有AI幻觉

综合评分：
- 高风险 (>70% AI特征)
- 中风险 (40-70%)
- 低风险 (<40%)
```

## 注意事项

1. **输入格式**：优先接收 PDF 文件或纯文本，URL 需验证有效性
2. **检测局限**：AI 检测结果为概率判断，不作为最终判定依据
3. **隐私保护**：用户上传的论文内容仅用于本次分析，不做存储
4. **多语言支持**：可处理中英文论文，其他语言需确认
