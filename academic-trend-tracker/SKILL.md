# Academic Trend Tracker - 学术热点追踪 Skill

## 定位

**账号定位**：AI科研学术情报站
**核心价值**：用AI追踪学术动态，数据驱动的学术洞察
**差异化**：不做纯搬运，加入AI分析和观点

---

## ⚠️ 发布原则（必须遵守）

### 1. 内容真实性原则

**核心要求**：绝不发布虚假内容！

- ❌ 不确定的消息不要发布
- ❌ 没有官方来源的内容不要发布
- ❌ 疑似谣言的热点不要蹭
- ✅ 发布前必须在网上搜索确认
- ✅ 优先使用权威来源（官方媒体、学术机构官网、顶刊官网）

**确认流程**：
```
1. 搜索相关话题
2. 确认至少有2-3个可信来源
3. 如来源矛盾，放弃发布
4. 确认为真后再生成内容
```

### 2. 封面图规范

- **比例**：3:4（竖版，小红书推荐）
- **尺寸**：1024x1024（image_gen 支持的最近比例，实际展示竖版裁剪）
- **风格**：科技感、悬念感
- **元素**：问号、数据图表、学术元素
- ⚠️ **必须使用 `image_gen` 工具生成**，禁止使用本地 `gen_daily_cover.py` 脚本
  - 若 `image_gen` 超时，重试1次；仍失败则告知用户，不降级为本地合成

### 3. 标题规范

- ⚠️ **标题必须是疑问、感叹或反问式，严禁陈述句**
- 须有强烈情绪钩子：惊讶感 / 反差感 / 好奇心三选一
- emoji 前置，结尾可带 🤔 / ！/ ？
- ✅ 正确：`😱 这篇论文被引2万次！它到底说了什么？`、`🔬 AI论文数量超医学了？数据来了！`
- ❌ 错误：`📚 斯坦福发布AI指数报告，中美差距缩小`（纯陈述，无情绪钩子）

### Step 4.5: ⚠️ 发布前检查（防止重复发布）

**必须执行！每次发布前都要检查。**

#### 检查方法

```bash
# 查看已发布笔记列表
npx xhs-mcp usernote list

# 或者搜索特定关键词
npx xhs-mcp search --keyword "关键词"
```

#### 检查流程

1. 提取当前内容的**关键词**（标题核心词、事件名等）
2. 执行 `npx xhs-mcp usernote list` 获取已发布列表
3. 遍历已发布笔记的标题，**模糊匹配**关键词
4. 如果发现相似标题（匹配度 > 60%），判定为**已发布**

#### 判断标准

| 匹配情况 | 判定 | 操作 |
|----------|------|------|
| 标题完全相同 | ✅ 已发布 | 跳过发布，告知用户 |
| 标题相似度 > 60% | ⚠️ 可能已发布 | 询问用户确认 |
| 无相似标题 | ❌ 未发布 | 继续发布 |

#### 示例

```
用户要求发布："同济大学论文造假"

检查结果：
- 已发布列表中有："同济大学Nature论文造假始末"
- 相似度：80% > 60%
- 判定：⚠️ 已发布过类似内容

操作：
❌ 跳过发布
📢 告知用户："此内容今天已发布过，请选择其他热点"
```

### Step 5: 发布

**确认未发布后，执行发布：**

```bash
# 使用 Node.js 脚本发布（避免 Windows 参数解析问题）
node publish_hot.js
```

**发布脚本模板：**

```javascript
const { spawn } = require('child_process');

const config = {
  title: '标题（不超过20字）',
  content: '正文内容...',
  media: 'e:\\WorkBuddyProjects\\generated-images\\封面图.png',
  tags: '标签1,标签2,标签3'
};

const xhsCli = 'e:\\node_global\\node_modules\\xhs-mcp\\dist\\xhs-mcp.cjs';

const child = spawn('node', [
  xhsCli,
  'publish',
  '-t', 'image',
  '-m', config.media,
  '--title', config.title,
  '--content', config.content,
  '--tags', config.tags
], { 
  stdio: 'inherit'
  // ⚠️ 重要：不要加 shell: true，会导致中文内容被截断！
});
```

## 踩坑经验

### JSON格式要求
- **tags字段**：必须用**数组格式** `["tag1", "tag2"]`，不能用字符串 `"tag1,tag2"`，否则 xhs-mcp 会错误分割标签
- **body字段**：正文内容，避免使用未转义的特殊字符

### 封面图
- ⚠️ **必须使用 `image_gen` 工具生成封面图，禁止本地脚本降级**
- hunyuan-image 超时时重试1次，仍失败则告知用户跳过封面
- image_gen 支持的尺寸为 1024x1024，不支持 1024x1360，生成后小红书会自动裁剪展示
- **封面图路径必须放入 selected_images 数组第一位**，否则发布时封面不会出现
- **标题严禁陈述句**：必须是疑问/感叹/反问式，需有悬念感和情绪钩子
